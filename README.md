## Conceptos / Concepts

### Broker

рҹҮӘрҹҮё Servidor HTTP en `127.0.0.1:7899`. Solo rutea mensajes y almacena estado en SQLite (`~/.zaipex-acc/acc.db`). No toma decisiones вҖ” es un bus de comunicaciГіn pasivo. Se auto-arranca con el primer agente. Limpia peers muertos cada 30 segundos.

рҹҮ¬рҹҮ§ HTTP server on `127.0.0.1:7899`. It only routes messages and stores state in SQLite (`~/.zaipex-acc/acc.db`). It makes no decisions вҖ” it's a passive communication bus. Auto-starts with the first agent. Cleans up stale peers every 30 seconds.

### Agentes / Agents

рҹҮӘрҹҮё Cada agente es una instancia de Claude Code (u otro agente MCP) con un rol asignado. Se registra con el broker al conectarse. Puede enviar/recibir mensajes, leer/escribir estado compartido, y ver quiГ©n mГЎs estГЎ conectado.

рҹҮ¬рҹҮ§ Each agent is a Claude Code instance (or any MCP agent) with an assigned role. It registers with the broker on connect. It can send and receive messages, read and write shared state, and see who else is online.

### Roles

рҹҮӘрҹҮё Definen la responsabilidad del agente. Cada rol recibe automГЎticamente un nombre de cientГӯfico famoso. `send_to_role` permite broadcast a todos los agentes con un rol dado.

рҹҮ¬рҹҮ§ Define what an agent is responsible for. Each role automatically gets a famous scientist's name. `send_to_role` broadcasts to every agent with that role.

### Estado compartido / Shared State

рҹҮӘрҹҮё Key-value store por namespaces. Ideal para contratos de API, esquemas, configuraciones. Persiste en SQLite вҖ” sobrevive reinicios.

рҹҮ¬рҹҮ§ Key-value store organized by namespaces. Great for API contracts, schemas, configs. Persisted in SQLite вҖ” survives restarts.

### Proyectos / Projects

рҹҮӘрҹҮё Agrupa agentes que trabajan juntos. Config en `~/.zaipex-acc/projects/<n>.json`. Datos aislados por proyecto.

рҹҮ¬рҹҮ§ Groups agents that work together. Config at `~/.zaipex-acc/projects/<n>.json`. Data isolated per project.
